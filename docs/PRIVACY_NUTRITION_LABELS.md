# Troodie App - Privacy Nutrition Labels Documentation
## For App Store Connect Privacy Questionnaire

Last Updated: January 2025

---

## Data Collection Overview

This document outlines all data types collected by the Troodie app to accurately complete the App Store Connect privacy questionnaire.

---

## 1. Contact Info
### Email Address
- **Collected**: Yes
- **Purpose**: Account creation, authentication, user identification
- **Linked to Identity**: Yes
- **Used for Tracking**: No
- **Storage**: Supabase Auth (secured database)

---

## 2. Identifiers
### User ID
- **Collected**: Yes (UUID generated by Supabase)
- **Purpose**: User account management, linking user-generated content
- **Linked to Identity**: Yes
- **Used for Tracking**: No
- **Storage**: Supabase database

### Device ID / Push Token
- **Collected**: Yes (when notifications enabled)
- **Purpose**: Push notifications delivery
- **Linked to Identity**: Yes
- **Used for Tracking**: No
- **Storage**: Supabase database (push_tokens table)

---

## 3. User Content
### Photos
- **Collected**: Yes (optional - profile photos, restaurant photos)
- **Purpose**: User profile customization, restaurant content
- **Linked to Identity**: Yes
- **Used for Tracking**: No
- **Storage**: Supabase Storage

### Text Content
- **Collected**: Yes
- **Types**: Restaurant notes, board descriptions, community posts, comments
- **Purpose**: App functionality - saving and sharing restaurant experiences
- **Linked to Identity**: Yes
- **Used for Tracking**: No
- **Storage**: Supabase database

---

## 4. Usage Data
### Product Interaction
- **Collected**: Limited (only essential app interactions)
- **Types**: 
  - Restaurant saves/unsaves
  - Board creation and management
  - Community interactions
- **Purpose**: App functionality
- **Linked to Identity**: Yes
- **Used for Tracking**: No
- **Storage**: Supabase database

---

## 5. Diagnostics
### Crash Data
- **Collected**: Yes (via Sentry)
- **Purpose**: App stability and bug fixing
- **Linked to Identity**: Yes (user ID for context)
- **Used for Tracking**: No
- **Third-Party Service**: Sentry.io

### Performance Data
- **Collected**: Yes (via Sentry)
- **Types**: App launch time, screen load times, API response times
- **Purpose**: Performance monitoring and optimization
- **Linked to Identity**: Yes (user ID for context)
- **Used for Tracking**: No
- **Third-Party Service**: Sentry.io

---

## 6. Location
### Precise Location
- **Collected**: No
- **Note**: Location permissions removed from app

### Coarse Location
- **Collected**: No
- **Note**: Restaurant locations come from Google Places API, not device location

---

## 7. Sensitive Info
- **Health & Fitness**: No
- **Financial Info**: No
- **Contacts**: No
- **Browsing History**: No
- **Search History**: Yes (in-app restaurant searches for functionality only)
- **Sensitive Info**: No

---

## Third-Party Services

### 1. Supabase (Database & Auth)
- **Data Shared**: Email, user content, usage data
- **Purpose**: Core app functionality, authentication
- **Privacy Policy**: https://supabase.com/privacy

### 2. Sentry (Error Tracking)
- **Data Shared**: Crash logs, performance metrics, user ID for context
- **Purpose**: App stability and performance monitoring
- **Privacy Policy**: https://sentry.io/privacy/

### 3. Google Places API
- **Data Shared**: Restaurant search queries
- **Purpose**: Restaurant data and search functionality
- **Privacy Policy**: https://policies.google.com/privacy

### 4. Expo (Development Platform)
- **Data Shared**: None in production
- **Purpose**: App building and deployment
- **Privacy Policy**: https://expo.dev/privacy

---

## Data Retention

- **User Account Data**: Retained until user deletes account
- **User Content**: Retained until user deletes specific content or account
- **Push Tokens**: Retained while notifications enabled, removed when disabled
- **Diagnostic Data**: 90 days (Sentry default)

---

## Data Deletion

Users can delete their account and all associated data through:
1. Settings > Delete Account (in-app)
2. Contact support at team@troodieapp.com

Upon account deletion:
- All user data is permanently removed from our databases
- Deletion includes: profile, saves, boards, communities, posts, comments, follows, push tokens
- Process is irreversible

## Notes for App Review

1. **No Advertising**: App does not display ads or use advertising SDKs
2. **No Tracking**: No cross-app or cross-site tracking
3. **No Location Services**: App does not request device location
4. **Minimal Data Collection**: Only essential data for app functionality
5. **User Control**: Users can delete all their data at any time
6. **HTTPS Only**: All data transmission uses encryption
7. **No Encryption**: App declares `ITSAppUsesNonExemptEncryption: false` (uses standard HTTPS only)
